# Contributing to GDevelop Academy

Thank you for your interest in contributing to GDevelop Academy! We appreciate your help and support in improving the project for everyone. Below are the guidelines for contributing to this repository:

## Types of Contributions
- **Bug Reports**: Identify and report any issues or bugs you find.
- **Feature Requests**: Suggest new features or enhancements.
- **Code Contributions**: Submit code improvements or added features.
- **Documentation**: Help us by improving existing documentation or writing new guides.

## How to Start
1. **Fork the Repository**: Create your own copy of the repository and make changes there.
2. **Clone Your Fork**: Clone your fork of the repository to your local machine:
   ```bash
   git clone https://github.com/YOUR_USERNAME/gdevelop-academy.git
   ```
3. **Create a Branch**: Create a new branch for your feature or fix:
   ```bash
   git checkout -b your-feature-branch
   ```

## Content Guidelines
- Ensure that your contributions are relevant to the project. 
- Follow the existing structure and guidelines in the repository.
- Provide clear and concise explanations for any new content.

## Style Guidelines
- Follow the code style provided in the repository.
- Use descriptive commit messages for your changes.
- Format your code according to the project's formatting rules.

## Issue Reporting
- Search existing issues to ensure that your issue hasn’t already been reported.
- Clearly describe the problem, including steps to reproduce, if applicable.
- Provide any relevant screenshots or logs when reporting.

## Review Process
- Once you’ve completed your changes, push them to your fork:
   ```bash
   git push origin your-feature-branch
   ```
- Submit a Pull Request (PR) to the main repository. Provide a description of your changes.
- Address any feedback or requested changes during the review process.

## Code of Conduct
We expect all contributors to follow the [Code of Conduct](CODE_OF_CONDUCT.md). Be respectful and inclusive in collaboration with others.

## Help Information
If you need assistance, feel free to reach out in the issues section or contact maintainers directly.
