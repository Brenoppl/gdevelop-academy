# Contributing to GDevelop Academy

We love your interest in contributing to GDevelop Academy! Please read this document to understand our guidelines and how you can help.

## How to Contribute

1. **Fork the repository**: Create a personal copy of the project by clicking the fork button.
2. **Clone your fork**: Use `git clone https://github.com/YOUR_USERNAME/gdevelop-academy.git` to clone your fork to your local machine.
3. **Create a new branch**: Use `git checkout -b your-feature-branch` to create a new branch for your changes.
4. **Make your changes**: Implement your feature, fix bugs, or improve the documentation.
5. **Commit your changes**: Use meaningful commit messages that explain the purpose of your changes.
6. **Push to your fork**: Push your changes back to your GitHub fork using `git push origin your-feature-branch`.
7. **Create a pull request**: Go to the original repository and create a pull request to propose your changes.

## Code of Conduct

Please make sure to read and follow our [Code of Conduct](CODE_OF_CONDUCT.md) when participating in the project.

## Reporting Issues

If you encounter any issues, please report them using the Issues tab in the repository. Be sure to provide as much detail as possible.
